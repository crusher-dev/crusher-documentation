---
title: Setup crusher on docker
description:
---

An OSS version of crusher is available on dockerhub, `crusherdev/crusher:oss1.0`, which can be used to run crusher
without much configuration.

This works with all major infra as docker is platform-agnostic.

**Note**: The `docker-compose.yml` below also creates mongodb, redis and mysql services since crusher doesn't come with it by default.

1) Create docker-compose.yml

```
version: '3'

networks:
  crusher-network:
    driver: bridge
    ipam:
      driver: default
      config:
        - subnet: 172.12.23.0/24

services:
  mongodb:
    image: 'bitnami/mongodb:latest'
    environment:
      - ALLOW_EMPTY_PASSWORD=yes
      - MONGODB_DATABASE=crusher
    networks:
      - crusher-network

  redis:
    image: 'bitnami/redis:latest'
    environment:
      - ALLOW_EMPTY_PASSWORD=yes
    networks:
      - crusher-network

  mysql:
    image: 'bitnami/mysql:latest'
    environment:
      - MYSQL_ROOT_PASSWORD=password
      - MYSQL_DATABASE=crusher
      - MYSQL_AUTHENTICATION_PLUGIN=mysql_native_password
    networks:
      - crusher-network

  crusher_oss:
    networks:
      - crusher-network
    image: crusherdev/crusher:oss1.0
    depends_on:
      - mysql
      - mongodb
      - redis
    environment:
      - DB_HOST=mysql
      - DB_USERNAME=root
      - DB_PASSWORD=password
      - DB_DATABASE=crusher
      - DB_PORT=3306
      - MONGODB_CONNECTION_STRING=mongodb://mongodb:27017/crusher
      - REDIS_HOST=redis
      - REDIS_PORT=6379
      - REDIS_PASSWORD=
      - BACKEND_SERVER_URL=http://localhost:3000/server
      - NEXT_PUBLIC_CRUSHER_MODE=open-source
      - NEXT_PUBLIC_INTERNAL_BACKEND_URL=http://localhost:3000/server
    ports:
      - "3000:3000"
    links:
      - "mysql:mysql"
      - "mongodb:mongodb"
      - "redis:redis"
    container_name: 'crusher_oss'
    stdin_open: true # docker run -i
    tty: true
```

2) Run docker-compose services
```
    docker-compose up -d --force-recreate --no-deps
```

3) Wait for 1-2 mins for db migrations to finish on your new setup.

You can also check the logs, to see when db migrations get completed using `docker logs -f crusher_oss`

<hr/>

Voila! Crusher is configured and ready to use. Try opening `http://localhost:3000`
